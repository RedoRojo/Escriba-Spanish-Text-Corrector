Por favor, revisa estas oraciones y dime si contienen errores. Sé sincero y detallado: identifica cualquier error de ortografía, gramática, sintaxis o estilo, corrígelo y explícame por qué era incorrecto. Si todo está correcto, indícalo también.

Clasifica cada error en una de estas categorías:
- ortografía: Errores de ortografía (ej: "havía" → "había")
- gramática: Concordancia de género/número, conjugación verbal (ej: "las niños" → "los niños")
- puntuación: Signos de puntuación incorrectos o faltantes, acentos diacríticos (ej: "el" → "él")
- semántica: Palabra incorrecta para el significado deseado (ej: "aludir" vs "eludir")
- estilo: Redundancia, frases poco claras, construcciones awkward (ej: "en el día de hoy" → "hoy")

Responde ÚNICAMENTE con un objeto JSON válido. Sin markdown, sin fences, sin texto adicional.

El JSON debe tener esta estructura exacta:
{
  "errors": [
    {
      "type": "ortografía",
      "original": "texto original con el error",
      "correction": "texto corregido",
      "explanation": "Explicación breve en español de por qué es un error"
    }
  ],
  "corrected_text": "Texto completo con todas las correcciones aplicadas",
  "summary": {
    "total": 0,
    "by_type": {
      "ortografía": 0,
      "gramática": 0,
      "puntuación": 0,
      "semántica": 0,
      "estilo": 0
    }
  }
}

Reglas:
- Si no hay errores, devuelve un array "errors" vacío y el texto original en "corrected_text".
- No inventes errores. Solo señala problemas reales.
- El summary.total debe ser la suma de todos los by_type.
